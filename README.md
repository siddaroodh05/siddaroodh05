### B â€” SVG icons (Simple Icons CDN)
If you prefer small clean icons (SVGs) you can use the Simple Icons CDN and control size via an HTML <img> tag (GitHub README supports mixed HTML + Markdown).

Example HTML (keeps icons small and aligned):
<p>
  <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript" title="JavaScript">
    <img src="https://cdn.simpleicons.org/javascript/F7DF1E" alt="JavaScript" height="28"/>
  </a>
  <a href="https://www.typescriptlang.org/" title="TypeScript">
    <img src="https://cdn.simpleicons.org/typescript/3178C6" alt="TypeScript" height="28"/>
  </a>
  <a href="https://reactjs.org/" title="React">
    <img src="https://cdn.simpleicons.org/react/61DAFB" alt="React" height="28"/>
  </a>
  <a href="https://www.docker.com/" title="Docker">
    <img src="https://cdn.simpleicons.org/docker/2496ED" alt="Docker" height="28"/>
  </a>
</p>

How the Simple Icons CDN URL works:
https://cdn.simpleicons.org/{slug}/{hexcolor}
- slug: lowercase library name (e.g., react, nodejs, python, aws)
- hexcolor: hex color code for the icon (no #). You can omit color to get default black.

## Tips & Notes
- Favorites: use badges (shields.io) when you want labels and colors. Use SVG icons when you want a compact, icon-only look.
- Size: use HTML <img height="..."/> to control icon height. Markdown image syntax doesn't support size attributes.
- Color: with Simple Icons CDN you can append the hex color (no #) to the URL. With shields.io you set the color in the URL.
- Logos: shields.io supports many official logos via the `logo=` parameter (use the logo name used by Simple Icons or Shields).
- Accessibility: always include alt text and link to the official docs/homepage.
